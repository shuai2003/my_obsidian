# 4. 认识T800的的joint（连杆）和link（关节）
## 4.1 Joint 和 Link  关节靠连杆连接在一起
Joint决定Link怎么动
学习T800不是先学“零件”，而是先学它的运动树。Link是骨头，Joint是骨头之间的活动连接。Joint决定Link能怎么动，所有Joint组合起来形成T800的运动系统。
### 4.1.1 Link（连杆）

就是：
- 头
- 躯干
- 手臂
- 小腿
- 大腿
这些**刚性零件**。
它有：
- 长度
- 质量
- 惯量
- 形状
### 4.1.2 Joint（关节）
连接两个 link 的地方。膝关节、肘关节、肩关节等
例如：
```
肩膀：

Torso
  |
 Joint
  |
UpperArm
```

肩膀这个位置就是 joint。它决定：UpperArm 可以怎么运动？

# 6. T800的状态参数
```
nq:机器人的位姿 qpos 的准确定义是**广义坐标**，它是"能完整描述系统当前几何状态所需的最少一组数字"——这组数字里**既包含位置信息，也包含姿态信息**，取决于具体是什么类型的关节： 广义坐标的数量 注意自由关节
nv:描述速度 每一个关节的速度 描述系统"速度"状态所需的变量个数 自由度数量，也叫速度维度  等于自由度数
nu:可控制电机的数量 表示这个系统有多少个可以被控制的执行器（电机、气缸等
通常 `nu ≤ nv`，因为不是每个自由度都有对应的驱动器
qpos: nq对应qos的长度  想表示机器人的姿态 需要多少个数值
qvel: nv对应qvel的长度 想表示机器人的速度需要多少个数值
ctrl control(机器人关节控制器): nu对应ctrl的长度 控制机器人需要给多少个电机发送指令
```

## 6.1控制机器人就是控制三个参数qpos、qvel、ctrl

下面编写pd控制策略、策略推理模型、高级算法 都是在修改和读取三个数组里面的数字        
一般先读取qpos姿态和qvel速度，可以得到机器人当前是什么样的姿态和什么速度，然后我们的模型再去修改ctrl里面的数据，可以完成对位置速度的观测，然后决定我下一次的控制。

# 8.1 pd控制   控制电机的力矩（力气）
控制机器人的关节，保持某一个姿态。
**能不能直接强行写它的关节角，电机关节角度设置120度** **？？**
1.直接修改关节角的操作也叫瞬移操作，在粗糙的仿真，可以这样做；在游戏中是可能可以的。
2.在真实的物理世界是不行的，所有的角度变化都是通过电机来控制的，电机是由电流电压来驱动线圈产生磁场来进行控制的，**它是不能瞬时移动的** 使用扭矩把关节拉到某个位置
3.**机器人领域最经典的一个关节的控制方式叫做pd控制
